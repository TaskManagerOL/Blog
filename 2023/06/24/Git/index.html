工作图

初始化（工作区）

  1. 使用 git init 初始化
  2. 新建 .gitignore文件来忽略不上传的文件
  3. 也可以在github上初始化之后，git clone

缓存区

git add

如果你想上传所有文件

setting to .

如果你想让你上传的文件有着相同的后缀

setting to *.(suffix)

提交区（本地git库/版本库）

git commit

-m “该次提交的提交信息”

远程仓库

git remote

-v 显示所有仓库

show [remote]显示远程仓库信息

add [origin(shortname) [url]] 添加远程版本库，shortname为远程库名字（一般默认origin），url为远程仓库地址

rm name 删除远程仓库

rename old_name new_name 修改仓库名

在使用远程仓库之前建议配置一个ssh连接 教程 – 菜鸟教程

分支

分支系统是版本控制系统中最为重要的部分，使用其意味着你可以从开发主线当中分出来一条支线，从而在不影响主线开发的同时继续开发

创建分支命令（无分支名时，命令效果为查看当前的本地分支）

git branch （branch_name）

切换分支命令

git checkout -(参数)（branch_name）

-b 创建新分支的同时进入分支

-d 删除该分支

git merge

合并分支至当前分支命令

提交 拉取

git push <远程主机名> <本地分支名>:<远程分支名>

提交操作 提交本地的git库 中的本地分支至远程分支

当然，一般来说本地分支和远程分支名字相同，所以可以省略冒号及其后面的部分

若本地和远程版本有差异，则可以使用–force进行强制推送（或者-f）

第一次上传可以使用-u以设置默认值之后就可以使用不带任何参数的git push了

git pull <远程主机名> <远程分支名>:<本地分支名>

将远程主机 的 远程分支拉过来与本地的分支合并

若远程分支是和当前分支合并，则可以省略冒号后面的部分

撤销 删除

git restore

单使用该命令会将工作区更改了且未上传缓存区的文件撤销更改

–staged 将上传缓存区的对应文件撤出，且不会撤销工作区文件的更改

git rm

单使用该命令会将工作区的对应文件删除

–cached 将缓存区中的对应文件删除

-r 用于遍历删除文件夹及其下的子文件夹

-f 用于删除文件夹

git reset

用于回退你上传到提交区(本地git库)的操作

–soft <提交信息/版本号>

只撤销该次git commit，并将文件回退到git add中

–mixed <提交信息/版本号>

撤销git commit和git add

–hard <提交信息/版本号>

将一切返回到上次git commit提交时的状态，包括工作区文件

查看

git status

查看当前的git状态

git log

查看提交区的历史提交信息

–oneline 使得提交信息较为简单紧凑

-nx 其中x为查看日志的数量

pull request

  1. fork你需要修改的仓库
  2. git clone将仓库置于本地
  3. 根据issue或者自身需求来修改代码
  4. git add 将修改后的代码add置于缓存区
  6. git commit -m “update introduction”确认上传文件无误的前提下将代码commit至提交区
  7. git push 将提交区的代码push至远程fork仓库
  8. 在远程自己的fork仓库提出pr
  9. 等待仓库所有者通过pr

fork & local仓库更新

  1. 在自己的fork库中Fetch and merge将他人的fork库更新至自己的fork库
  2. cd进自己的工作区
  3. git fetch -all将自己fork库的修改更新到自己的本地git库
  4. git status查看自己工作区状态
  5. git pull拉取自己fork库的更改到本地
  6. 将自己的修改置于工作区
  7. git status查看自己的工作区状态
